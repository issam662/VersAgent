# Development Overview

This section provides guidelines for developers contributing to the PFE Project.

## Local Development Workflow

1.  **Branching**: Create a feature branch from `main` (e.g., `feat/auth-fix`).
2.  **Environment**: Ensure `.env` files are updated with local secrets.
3.  **Development**: Run the client and server concurrently using the `npm run dev` commands provided in the Quickstart.
4.  **Logs**: Monitor `client.log`, `server.log`, and `agent.log` for real-time debugging information.

## Debugging Tips & Common Pitfalls

- **Port Conflicts**: If `3000` or `5173` are taken, update the `PORT` env variable in your `.env` file.
- **Agent Timeouts**: Agents may take a long time to "think." If the request hangs, check the `agent.log` for reasoning loops.
- **Database Connections**: Ensure the database is reachable before starting the `server`.

## Test Strategy

- **Unit Tests**: Use Vitest for frontend components and Jest for backend services.
- **Integration Tests**: Focus on the API surface and the Agent-to-Tool communication.
- **Manual QA**: Every new feature should be verified in the `development` environment before merging.

## Request Tracing

To trace a request end-to-end:
1.  Capture the **Request ID** from the client.
2.  Search for this ID in `server.log` to see API handling.
3.  Locate the **Task ID** generated by the server.
4.  Search for the Task ID in `agent.log` to see the reasoning steps and tool outputs.
